# DomJudge Deployment

1. Install normally according to the [official manual](https://www.domjudge.org/docs/manual/9.0/index.html)
2. Remove the default nginx page from `/etc/nginx/sites-enabled/default`
3. Change the FPM upload parameters in `/opt/domjudge/domserver/etc/domjudge-fpm.conf`
4. Add the following to `/etc/mysql/my.cnf`

```
[mysqld]
max_connections=1000
innodb_log_file_size=10GB
max_allowed_packet=2GB
slow_query_log=1
slow_query_log_file=/var/log/mysql_slow.log
net_read_timeout = 7200
net_write_timeout = 7200

[mysqldump]
max_allowed_packet = 1G
```

5. Install redis and switch sessions to redis according
   to [this blog post](http://403f.cafe/p/icpc-2025-regional-review/#%E4%BC%98%E5%8C%96php-session%E5%82%A8%E5%AD%98)

# JudgeHost Deployment

Use the following script to generate the Docker Compose file.

{{% details summary="JudgeHost Docker Compose generation script" %}}

```shell
#!/usr/bin/env bash
set -euo pipefail

N="${1:-20}"

cat <<'YAML'
services:
YAML

for i in $(seq 1 "$N"); do
  if [[ "$i" -eq 1 ]]; then
    cat <<YAML
  judgehost-$i:
    image: docker.1ms.run/domjudge/judgehost:9.0.0
    hostname: judgedaemon-$i
    restart: unless-stopped
    privileged: true
    environment: &judgehost_env
      DOMSERVER_BASEURL: "http://<URL HERE>/"
      JUDGEDAEMON_PASSWORD: "<PASSWORD HERE>"
      DAEMON_ID: "$i"
    volumes:
      - /sys/fs/cgroup:/sys/fs/cgroup:rw
    networks:
      - domjudge-net

YAML
  else
    cat <<YAML
  judgehost-$i:
    image: docker.1ms.run/domjudge/judgehost:9.0.0
    hostname: judgedaemon-$i
    restart: unless-stopped
    privileged: true
    environment:
      <<: *judgehost_env
      DAEMON_ID: "$i"
    volumes:
      - /sys/fs/cgroup:/sys/fs/cgroup:rw
    networks:
      - domjudge-net

YAML
  fi
done

cat <<'YAML'
networks:
  domjudge-net:
    external: true
    name: domjudge-judgehosts_default
YAML
```

{{% /details %}}

**It is better not to modify judgehosts during the contest, or judge tasks may be lost.**

# Natsume Deployment

> Only for Natsume versions later than 0.1.2

1. Generate a CA certificate with the following commands

```shell
 openssl ecparam -name prime256v1 -genkey -noout -out ca.key
 openssl req -new -x509 \
     -key ca.key \
     -sha256 \
     -days 365 \
     -out ca.cert \
     -addext "basicConstraints=critical,CA:TRUE" \
     -addext "keyUsage=critical,keyCertSign,cRLSign" \
     -addext "subjectKeyIdentifier=hash"
 openssl pkcs8 -topk8 -inform PEM -in ca.key -outform PEM -out ca_pkcs8.key -nocrypt
```

2. Generate a TLS certificate for the caddy reverse proxy,
   with [this script](https://github.com/4o3F/Natsume/blob/main/assets/generate_cert.sh)
3. Preprocess the data with the Natsume data preprocessor
4. Create team categories (Participants, Girls, etc.)
5. Import the data generated by the Natsume data preprocessor: `organizations.json` first, then `teams.json`, and `accounts.yaml`
   last
6. Install parallel-ssh in advance

# Monitoring & Performance Tuning

1. Build nginx with Brotli according
   to [this blog post](https://403f.cafe/p/icpc-2025-regional-review/#%E4%BC%98%E5%8C%96%E5%AE%BD%E5%B8%A6%E5%B0%96%E5%B3%B0)
2. Install Redis and switch the DomJudge session store to Redis according
   to [this blog post](https://403f.cafe/p/icpc-2025-regional-review/#%E4%BC%98%E5%8C%96php-session%E5%82%A8%E5%AD%98)
3. Put opcache monitor's `opcache.php` in `/opt/domjudge/status/opcache.php`
4. Add the following to `/etc/nginx/nginx.conf` inside the http block

```
# DOMjudge scoreboard cache
fastcgi_cache_path /var/cache/nginx/domjudge
   levels=1:2
   keys_zone=dj_scoreboard:20m
   inactive=30s
   max_size=200m
   use_temp_path=off;
```

5. Create the cache path with the following commands

```shell
mkdir -p /var/cache/nginx/domjudge
chown -R www-data:www-data /var/cache/nginx/domjudge
```

6. Edit `/opt/domjudge/domserver/etc/nginx-conf-inner`; it should become like the following
